Palavra-passe

☁️ Sincronizar entre dispositivos

Opcional e desligada por omissão. Quando ativas, o teu roupeiro é cifrado neste dispositivo e só depois enviado. O servidor guarda um bloco de texto que não consegue ler — nem peças, nem fotos, nem nomes.

⚙️ Configuração inicial (uma vez só)
  1. Cria conta grátis em supabase.com e um projeto novo.
  2. No projeto: SQL Editor → cola e corre o bloco abaixo.
  3. Em Project Settings → API, copia o Project URL e a chave anon public.
  4. Cola-as nos campos aqui em baixo.
create table roupeiros (
  user_id uuid not null default auth.uid()
    references auth.users on delete cascade,
  perfil text not null,
  nome text, emoji text,
  salt text not null,
  iteracoes int not null default 600000,
  verificador jsonb not null,
  blob text not null,
  versao int not null default 1,
  atualizado timestamptz not null default now(),
  primary key (user_id, perfil)
);

alter table roupeiros enable row level security;

create policy "so o dono le"     on roupeiros for select
  using (auth.uid() = user_id);
create policy "so o dono insere" on roupeiros for insert
  with check (auth.uid() = user_id);
create policy "so o dono altera" on roupeiros for update
  using (auth.uid() = user_id);
create policy "so o dono apaga"  on roupeiros for delete
  using (auth.uid() = user_id);

As políticas acima garantem, do lado do servidor, que ninguém lê as linhas de outra pessoa — mesmo com a chave anon, que é pública por design.

Para não teres de fazer isto em cada dispositivo, escreve o URL e a chave em NUVEM_PADRAO, no topo de nuvem.js: aí qualquer dispositivo que abra a app já vem ligado.

Normalmente não precisas de mexer aqui: a sessão abre sozinha quando entras com o email e a palavra-passe da tua conta. Isto é a rede de segurança para quando falha.
